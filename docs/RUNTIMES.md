# 언어 런타임 선택

이 레포는 `uv`(Python), `node`(JS), `rustup`(Rust) 조합을 기본으로 둡니다.

## 선택 기준

- **속도와 단순성**: 설치/업데이트가 빠르고 설정이 단순한 도구 선호
- **표준성**: 커뮤니티에서 널리 쓰이고 문서가 풍부한 도구 선택
- **문제 분리**: 언어별 도구를 분리해 문제 원인을 좁히기 쉽게

## Python: uv + ruff + ty

- `uv`는 가볍고 빠르며, venv/툴 설치를 간단하게 관리할 수 있습니다.
- `ruff`는 린트/포맷 속도가 빠르고 설정이 단순합니다.
- `ty`는 LSP를 제공해 타입 체크 기반 편집 경험을 제공합니다.

## JavaScript: node + biome

- `node`는 기본 런타임으로 충분하고, 필요하면 버전 매니저로 확장 가능합니다.
- `biome`는 포맷/린트 속도가 빠르고 설정이 단순합니다.

## Rust: rustup

- `rustup`은 Rust의 표준 설치/버전 관리 도구입니다.
- `rustfmt`, `clippy`는 기본 컴포넌트로 추가합니다.

Rust는 `rustup-init`을 Homebrew로 설치한 뒤,
`run_onchange_install-brew-packages.sh`에서 `rustup-init -y`로 초기화합니다.

## 왜 mise/asdf를 쓰지 않았나

- 각 언어의 전용 도구가 더 빠르고 안정적인 경우가 많습니다.
- 설정 복잡도를 낮추고, 문제가 생겼을 때 원인을 좁히기 쉽습니다.
- 필요할 때 언어별로만 확장할 수 있습니다.

## 언제 mise를 고려할까

- 여러 언어 버전을 프로젝트별로 자주 바꿔야 할 때
- 팀 표준이 mise/asdf로 통일되어 있을 때

그런 경우에는 `mise`를 도입하고, zsh 초기화와 `.mise.toml`만 추가하면
쉽게 전환할 수 있습니다.

## 대안 선택 가이드

- Python에서 여러 버전을 자주 바꾼다면 `pyenv`/`mise` 고려
- Node 버전 전환이 잦다면 `fnm`/`nvm` 고려
- 팀 표준이 있다면 그 표준을 우선
