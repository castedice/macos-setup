# direnv

`direnv`는 디렉터리별로 환경 변수를 자동 로드합니다.

## 왜 사용하는가

- 프로젝트마다 다른 환경 변수를 자동으로 적용해 실수를 줄입니다.
- `.envrc`만으로 팀 내 설정 흐름을 문서화할 수 있습니다.
- 쉘 시작 시점이 아니라 **디렉터리 진입 시점**에 적용되어 가볍습니다.

## 기본 흐름

1) 프로젝트 루트에 `.envrc` 작성  
2) `direnv allow` 실행  
3) 해당 디렉터리에 들어갈 때 자동 적용

## uv와 함께 쓰기

이 레포는 `~/.config/direnv/direnvrc`에 `use_uv` 함수를 제공합니다.

예시 `.envrc`:

```sh
use_uv
```

`use_uv`는 `.venv`가 없으면 `uv venv`로 생성하고 자동 활성화합니다.

## 보안 가이드

- `.envrc`는 쉘 스크립트로 실행됩니다. 신뢰하지 않는 레포에서는
  `direnv allow`를 바로 실행하지 마세요.
- 비밀키는 `.envrc`에 직접 넣지 말고 별도 파일/비밀 관리 도구로 분리하세요.

## 커스터마이징

- 팀 공통 변수를 `.envrc`에만 유지
- 개인 전용 변수는 로컬 파일로 분리 후 source
